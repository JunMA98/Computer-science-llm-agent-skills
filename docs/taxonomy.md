# Taxonomy

This public repository uses a workflow-first taxonomy for CS research and programming work.

| Category | Scope | Example skills |
| --- | --- | --- |
| Research Planning & Task Decomposition | scope setting, problem framing, milestones, risk planning | `research-planning` |
| Literature Review & Reference Management | paper discovery, synthesis, citations, library automation | `literature-review`, `citation-management`, `pyzotero` |
| Paper Writing & Revision | manuscript drafting, section rewriting, venue-oriented revision | `scientific-writing`, `code-to-paper` |
| Review, Rebuttal & Scholarly Evaluation | peer review, reviewer response, critical reading, scoring frameworks | `peer-review`, `reviewer-response`, `scholar-evaluation` |
| Benchmarking, Reproducibility & Experiment Ops | evaluation design, run tracking, code reproduction, ablations | `benchmark-design`, `experiment-tracking`, `code-reproduction` |
| Technical Writing & Artifact Conversion | markdown-first docs, diagrams, conversions, structured technical communication | `markdown-mermaid-writing`, `markitdown` |
| Data Engineering, Statistics & Visualization | data pipelines, EDA, statistical testing, figures, dashboards | `dask`, `polars`, `statsmodels`, `scientific-visualization` |
| ML / LLM / Graph / RL Workflows | model training, fine-tuning, explainability, graph ML, RL baselines | `transformers`, `scikit-learn`, `torch-geometric`, `stable-baselines3` |
| Agent Systems & Automation Development | tool-using agents, orchestration loops, evaluation for agent behavior | `agent-coding` |
| Presentation & Publication Assets | slides, posters, schematics, final presentation artifacts | `scientific-slides`, `latex-posters`, `scientific-schematics` |
| Repository Maintenance & Open-Source Release | repo cleanup, public release prep, docs and legal readiness | `repo-cleanup`, `github-release-prep` |
| Mathematical & Algorithmic Tools | symbolic math, graph analysis, algorithm support | `sympy`, `networkx` |

## Classification rule

A skill belongs in the public repository only when it directly supports recurring work for a CS researcher or research-oriented developer and does not currently have blocking scope, licensing, or maintainability issues.
